Legacy of Blade (2023)

Gameplay Programmer | 6 Months

About This Project

Legacy of Blade is a third-person action combat project I developed in Unreal Engine 4.27 using C++ and Blueprints. The focus was to build a responsive melee combat system and an axe recall mechanic inspired by God of War, while showcasing AI behavior, player interaction, and technical gameplay systems.

Project Info

  • Role: Sole Gameplay Programmer
  • Duration: 6 Months (Self-Directed)
  • Challenge: Implementing Procedural Generation using C++.
  • Status: Completed Prototype

Core Systems & Mechanics

Combat System

The combat system is built entirely using C++ and Blueprints, combining animation-driven logic, collision detection, and reactive enemy AI. Each weapon uses a dedicated Box Collision Component, which activates only during the correct animation frames to ensure accurate hit registration and prevent false overlaps.

Enemy behavior is managed through a Finite State Machine (FSM) written in C++, handling states like Idle, MoveToTarget, Attack, and Dead. This guarantees that enemies perform one clear action at a time and react intelligently to player attacks.

Technical features:
  • Timers control attack intervals, recovery, and combo chains.
  • Montage sections sync animation events with active hit windows.
  • Particle and sound triggers enhance real-time combat feedback.
  • Modular design allows easy integration of new weapons or enemy types.

This system delivers fast, reliable, and visually responsive melee combat with clean gameplay architecture.

Weapon System (Sword & Leviathan Axe)

I developed a versatile weapon system in Unreal Engine (C++) that supports both close-range sword combat and a throwable-recallable Leviathan Axe inspired by God of War. This project demonstrates strong understanding of actor components, collision handling, and math-driven weapon mechanics.

Key Technical Features:
  • Axe Throw: Uses FMath::Lerp for smooth position interpolation and continuous rotation for realistic spin visuals with spatial flight audio.
  • Axe Recall: Implemented a custom BoomerangeCurve() using a Quadratic Bézier Curve (P = (1-t)²P₀ + 2(1-t)t*P₁ + t²P₂) for natural boomerang-like return motion.
  • Crosshair-Based Aiming: Calculates direction using PlayerController->DeprojectScreenPositionToWorld() for accurate 3D targeting.
  • Collision Control: The UBoxComponent (CombatCollision) remains disabled by default and activates only during attack or throw using ActivateCollision(), ensuring precise hit detection.

This system blends physics, math, and animation into responsive, cinematic weapon gameplay.

Maheshwara Actor (Technical Details)

The Maheshwara Actor is a fully scripted C++ interaction system that handles the process of unlocking and spawning the Leviathan Axe based on player progression. It uses a Sphere Collision Component to detect when the player enters its range and communicates directly with the Player character class to check the current coin count. Depending on the result, it dynamically plays the appropriate dialogue using UGameplayStatics::PlaySound2D() and displays a Widget prompt for interaction. When the player holds the 'E' key long enough, the system validates the input, plays a final dialogue sequence, triggers Niagara particle effects, and spawns the Leviathan Axe actor into the world through SpawnActor(). Timers control dialogue pacing and actor lifetime, while an AudioComponent manages continuous voice playback for immersive feedback.

Object Interaction Section

I developed a set of dynamic, reusable world-interaction objects. These components were built primarily in C++ and use Overlap Events and C++ Interfaces to ensure they can affect both the player and enemies in a modular way.

  • Bomb: I set up a simple overlap system so that when the player or even an enemy steps on it, the bomb explodes and applies damage to nearby actors. It was my first time handling area-based damage, and it felt great to see both sides react to it.
  • Coins: Every time the player collects a coin, the value updates instantly on the HUD. These coins aren’t just for show — they’re used later to unlock the Leviathan Axe from Maheshwara.
  • Spike Trap: This trap hurts both enemies and players. When triggered, spikes shoot up through an animation, deal damage, and then retract — simple but satisfying to see it working.
  • Healing Drink: It restores 25 health when collected, but only if the player isn’t already at full health.
  • Auto Door: This was my favorite to make. After learning Interpolation (FMath::Lerp), I wanted to try something fun, so I built a door that smoothly opens upward when the player steps on a pressure stone, stays open for a few seconds, and then closes automatically. It was the first mechanic where I combined collision detection, two static meshes, and movement interpolation — and it really made the level feel dynamic.

HUD & UI System

I implemented a complete HUD and UI system in my 3D sword-fighting project using a mix of C++ and Unreal Engine Widgets. The gameplay HUD updates dynamically based on the player’s actions, while the frontend menus manage game flow and settings.

For the enemy health bar, I created a floating widget that only appears when the player is close to an enemy. Its position is updated every frame by converting the enemy’s world position into screen space using ProjectWorldLocationToScreen(). After calculating the screen coordinates, I adjust the Y-offset and set the widget’s position and size in the viewport, creating a clean floating health bar that follows the enemy smoothly.

For interaction prompts like the ‘E’ Key icon near the Maheshwara actor, I used a Widget Component attached directly to the actor. This allows the prompt to exist in 3D space and appear only when the player enters the interaction range.

In addition, I created Main Menu, Pause Menu, and Settings/Resolution Menu, forming the project’s complete frontend UI system.

Challenges & Solutions

Levithan Axe:-

When I was working on the Leviathan-style axe, one of the biggest problems I faced was getting the axe return movement, the player’s recall animation, and the axe recall sound to match perfectly. Because all three started at the same time, the timing would break depending on the axe’s distance. If the axe was close, it reached too early. If the axe was far, it reached too late. The result was a mismatch every time.

To fix this, I redesigned the system so the axe always returns in a fixed duration, matching the length of the recall animation. My recall animation is 0.7 seconds, so I made the axe movement always complete in exactly 0.7 seconds, using time-based interpolation instead of speed. This ensured perfect sync, no matter how far the axe is.

Boss Enemy:-

When creating the Boss enemy, I wanted it to feel completely different from normal enemies—both visually and in combat behavior. I chose the Buff_Red character from Paragon Minions because it looked powerful and aggressive, which fit the role perfectly.

One of the biggest technical challenges was attack Collision detection. The boss uses four attacks—left punch, right punch, kick, and a heavy smash. Initially, I added separate Box Collisions to each hand and leg, but this caused unnecessary performance overhead. To solve this, I redesigned the system using one Box Collision and attached it dynamically to different sockets on the skeletal mesh (left hand, right hand, and leg) depending on which attack animation was playing. This simplified logic, improved performance, and kept hit detection accurate.

Another major challenge was the kick animation. The Paragon pack didn’t include any kick animations, so I downloaded one from Mixamo—but its skeleton didn’t match the Paragon skeleton at all. The animation looked completely broken. I learned about retargeting, mapped the Mixamo skeleton to Unreal’s Humanoid rig', then retargeted that animation to the Paragon skeleton. After adjusting pelvis height, root motion, and fixing twisting bones, the kick animation finally played correctly.
